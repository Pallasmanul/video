


* 从零实现一个小智服务器

在如今AI爆火的时代，出现了一个出圈的产品,AI小智。凭借着出色的语音聊天能力，丰富的MCP服务，低成本的硬件门槛，爆火网络。


在大家忙于DIY自己的小智硬件的时候， 让人不免有些好奇， 小智AI的后端是如何工作的呢？


小智最核心的功能就是接收用于的语音数据， 将语音音频数据转换为文本信息，交给AI大模型之后，将AI大模型回复的内容再转换成音频，交给设备播放。

听起来很简单对不对。

但在实际过程中，涉及到很多技术细节， 比如设备与服务器如何通信，传输过程中数据的压缩和解压， 加密解密,  如何动态控制设备,  短期和长期记忆等。


所以后端服务器才是小智AI的核心。


当然，目前github上已经出现了非常多优秀的小智AI后端项目, 但是这些项目大多基于websocket协议，与官方基于MQTT和UDP协议的后端服务明显不同。


所以, 在接下来的一系列视频中, 我会从零搭建一个基于MQTT和UDP的本地化小智服务器, 以此来实现一个功能相对完善的小智服务器。当然，在笔记本上本地部署服务, 同时处理小智后端的所有任务,  性能和质量肯定会有所差距,  请各位大佬轻喷。
