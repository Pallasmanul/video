


* 从零实现一个小智服务器

随着AI时代的来临， 各种AI产品犹如雨后春笋般涌现。

AI小智，凭借出色的语音能力,丰富的MCP服务和低成本的硬件, 
俘获了大批DIY爱好者。


在大家忙于DIY自己的小智硬件的时候， 让人不免有些好奇， 小智AI的后端是如何工作的呢？

其背后的运行逻辑非常简单。
设备接收到音频之后，将音频传输给后端，
后端接收到用户语音, 转成文字提交给AI大模型, 再把AI的回复转换成音频交由设备播放。

听起来很简单对不对。

但是要系研究起来，这里面门道不少?
比如设备与服务器如何通信，传输过程中数据的压缩， 加密,  如何控制设备执行指令。


所以后端服务器才是小智AI的核心。

当然，目前github上已经出现了非常多优秀的小智AI后端项目, 但是这些项目大多基于websocket协议，与官方基于MQTT和UDP协议的后端服务明显不同。


所以, 在接下来的一系列视频中, 我会从零搭建一个基于MQTT和UDP的本地化小智服务器, 以此来实现一个功能相对完善的小智服务器。当然，在本地部署服务, 同时处理所有小智后端任务,  性能和质量将较于官方肯定会有所差距,  请各位大佬轻喷。
