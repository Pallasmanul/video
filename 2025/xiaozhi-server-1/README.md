首先，小智设备固件有存在配置或者不存在配置俩种情况

如果小智设备不存在设备, 则小智会直接连接服务器的OTA服务, 通过OTA服务获取注册信息。



这里大家可能会疑惑, OTA服务是一个设备固件升级服务,  为什么会和设备配置有关?
这里藏着小智设备"偷偷配置"的巧思! 通过分析小智源码, OTA服务不仅能传输固件升级包, 还负责关键的设备配置。

听着有点混乱对不对。 举个例子来说。 没有配置的小智设备是刚入职的"萌新员工", 需要跑去 OTA服务这个 "人事部" 报到.

'人事部' (OTA服务器) 接到 "萌新" 递来的设备信息简历后, 通过查询对应的档案, 知道这个是新来的员工, 给设备发个"激活码工牌",  设备这个萌新员工拿到这个激活码工牌之后, 大声的念出激活码工牌号。 咱们用户就像是领导, 将对应的激活码输入到对应账号下的智能体, 就完成了员工的入职。 设备"萌新员工" 入职后, 将可以拿到 工作证 , 可以愉快的通信啦。

对应是有配置的小智设备是"已入职员工" , 跑去 OTA服务这个 "人事部" 报到, 人事部会查询档案, 知道这个是老员工, 直接给出"公作证"
